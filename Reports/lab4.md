# 实验四: 目标代码生成 实验报告

## 1. 编译方式

使用 `make` 进行编译, 执行

``````bash
cd ./Code/
make
``````

## 2. 功能介绍

接受输入文件和输出文件两个参数, 将源代码编译成MIPS32目标代码后输出.

``````bash
./parser test.cmm test.s
``````

## 3. 技术细节

### 3.1 中间代码预处理

上一阶段生成的中间代码是链表形式, 为方便后续翻译和基本块划分, 先将其转换成数组形式. 

同时根据中间代码生成的特点, DEC指令分配空间后必定会紧跟着一条ADDRESS指令对变量取地址, 所以这里直接将这两条指令合并为一条DEC指令, 参数为空间大小, 将首地址存入目标变量. 同时由于不存在全局变量, C--也不支持指针, 所以ADDRESS指令将不会再出现.

然后将整个中间代码划分成不同的函数, 函数内再划分成各个基本块进一步处理.

此处对前一阶段的函数调用的翻译进行了调整, 使得ARG指令和CALL指令必定在一起, 也就是一个函数调用必定是若干个连续的ARG指令后紧跟着一个CALL指令. 如此处理后ARG指令和CALL指令就可以合起来单独划分成一个基本块处理.

### 3.2 变量名映射

为方便处理, 在每个函数块内, 将变量名映射到一个唯一的索引.

``````c
typedef struct VarMap{
    const char* var_name;
    int idx;
} VarMap;

VarMap var_map[MAX_VAR];
int var_map_size = 0;

int get_var_idx(const char* var_name);
``````

### 3.3 寄存器分配算法

采用的局部寄存器分配算法, 也就是在一个基本块开始时所有寄存器都空闲, 然后每次分配空闲的寄存器, 当没有空闲寄存器时将下一次使用时刻最晚的寄存器溢出到内存, 在基本块结束时将所有非空闲寄存器都溢出到内存. 同时, 每翻译完一个语句, 都会更新当前分配的寄存器的next_use, 如果不再使用会立即释放.

``````c
typedef struct RegDesc {
    int is_free;
    int var_idx;
    int next_use; // 寄存器中变量的下一次使用位置, -1表示未计算
} RegDesc;

typedef struct VarDesc {
    int reg_idx; // 变量所在寄存器, 寄存器free时不会清除该项, 只有在被其他变量占用时才会置-1.
    int slot; // 局部变量槽, 本质就是第几个局部变量
} VarDesc;

RegDesc regs[REG_NUM];
VarDesc vars[MAX_VAR];

void init_allocation();
int allocate(const char* var_name);
int ensure(const char* var_name);
void free_if_unused(int cur_ir_idx, int block_end, const char* var_name);
``````

### 3.4 函数调用与栈结构

栈结构如下:
``````
high
+-----------------------+
| caller frame          |
+-----------------------+
| ...                   |
| arg6                  |
| arg5                  |
+-----------------------+ <- $fp
| saved $ra             |
+-----------------------+
| old $fp               |
+-----------------------+
| slot 0                | ($fp - 12)
| slot 1                |
| ...                   |
+-----------------------+
| dynamic (args)        |
+-----------------------+ <- $sp
low
``````

函数序言中会一次性分配栈空间, 由于大小需要翻译完整个函数体才能知道, 所以采用的回填策略. 

对于函数参数, 无论参数是放在寄存器上还是栈上, 都会全部取出并放到形参对应的栈内存上. 

`return` 指令会翻译成设置`v0`寄存器后直接跳转到一个固定标签`ret_函数名` , 函数尾声会设置这个标签并恢复栈结构.

对于函数调用指令 `CALL` , 当参数多于4个时, 会临时分配栈空间用来传递多余的参数, 调用结束返回后会恢复这部分栈空间.

### 3.5 其他

目标代码的开头是固定的, 声明一个全局符号`main` , 定义`read`和`write`两个函数, 具体内容位于全局变量`begin_asm`中. 这部分会直接拷贝到文件开头.

设定的可供自由分配的寄存器是19个, 分别是`t0-9`, `s0-7` 和 `v1`.

对于运算指令, 当两个操作数都是立即数时, 会直接计算出结果使用`li`指令直接加载这个结果.
